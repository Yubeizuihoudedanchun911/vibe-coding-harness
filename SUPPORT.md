# Support

Vibe Coding Harness is maintained as an open-source project on a best-effort
basis. Choose the route that matches your request so maintainers and other
contributors can respond efficiently.

## Choose the right route

- For installation, configuration, CLI usage, migration, recovery, or Provider
  integration questions, open a
  [support request](https://github.com/Yubeizuihoudedanchun911/vibe-coding-harness/issues/new?template=support_request.yml).
- For reproducible incorrect behavior, open a
  [bug report](https://github.com/Yubeizuihoudedanchun911/vibe-coding-harness/issues/new?template=bug_report.yml).
- For a proposed product or contract improvement, open a
  [feature request](https://github.com/Yubeizuihoudedanchun911/vibe-coding-harness/issues/new?template=feature_request.yml).
- For a suspected vulnerability, do not open an issue. Follow
  [SECURITY.md](SECURITY.md) and use
  [GitHub private vulnerability reporting](https://github.com/Yubeizuihoudedanchun911/vibe-coding-harness/security/advisories/new).

## Before opening a request

1. Read [README.md](README.md) and search existing issues and pull requests.
2. Reproduce the problem on a supported Python version and the latest
   revision when possible.
3. Remove credentials, private repository content, personal data, and other
   sensitive information from commands, state, logs, and screenshots.

## What to include

Provide the exact Vibe revision or release, operating system, Python version,
Codex CLI version when relevant, sanitized command, expected outcome, observed
outcome, and the smallest safe evidence needed to understand the request.

Public support has no guaranteed response time or service-level agreement.
Maintainers may redirect, close, or convert requests that use the wrong route
or cannot be investigated safely.
